# 备忘录功能升级计划

## 目标
将备忘录功能从本地 JSON 存储迁移到用户指定的飞书多维表格，并实现定时推送提醒。

## 用户需求
1. 数据存储到飞书多维表格：`https://ycse0qxiay.feishu.cn/base/LbLub4mKAaZfhUs5wxmcw2AJnyg?table=tbllqP1t5Gzphj5C&view=vewOGWGRcW`
2. 内容字段自动识别分类（合同流程、付款流程等）
3. 每天 9:00、12:00、14:00、15:00、16:00、17:00、18:00 推送未完成事项
4. 支持触发词：备忘录、备忘、memo

## 步骤

### 第一步：识别飞书多维表格结构
- 调用飞书 API 获取表格字段列表
- 确认字段：提醒事项、内容、状态、日期等
- 确认 appId/appSecret 已验证可用

### 第二步：修改 memo.py 脚本
- 新增 `fields` 配置：表格 app_token、table_id
- 修改 `cmd_add` 函数：
  - 调用飞书 API 写入多维表格
  - 自动识别内容分类（合同流程/付款流程）
  - 自动填充当天日期
- 修改 `cmd_send` 函数：
  - 查询状态不为"已完成"的记录
  - 推送飞书卡片消息
- 修改 `cmd_list` 函数：
  - 从飞书表格读取数据

### 第三步：修改 SKILL.md
- 更新数据存储说明（飞书多维表格）
- 更新触发词和使用流程
- 更新时间规则（新的推送时间）

### 第四步：配置定时任务
- 使用 launchd 调度定时推送
- 时间点：9:00、12:00、14:00、15:00、16:00、17:00、18:00

### 第五步：测试验证
- 测试添加备忘录（触发词 + 自动分类）
- 测试列表读取
- 测试推送功能
- 验证写入飞书表格成功
